# Phase 7: launchd Cron + Notifications — Implementation Plan

> Same execution pattern as prior phases. Thin slice.

**Goal:** Every weekday at 6 AM, the Mac runs the full pipeline (ingest → dossier → notify) automatically. The operator wakes up to a macOS notification banner saying "Today's plans are ready" and opens the dashboard to review.

**Architecture:**
- `workers/daily_run.py` — orchestrator that runs ingest scripts + dossier builder for every client that has OAuth credentials. Skips clients without creds with a clear log line. Logs to `~/Library/Logs/trak-automations/daily.log`.
- macOS native notification via `osascript display notification` (no third-party API, no Pushover account needed). Sends one banner per successful run.
- launchd plist at `~/Library/LaunchAgents/com.trakautomations.daily-run.plist`. Runs at 6 AM Mon-Fri.
- `scripts/install_launchd.sh` — operator runs once, generates the plist with the operator's actual Python venv path, loads it.

**Out of scope:**
- Pushover / email / Slack (operator can wire later — `osascript` is fine for solo use)
- Per-client cron customization (one schedule for all)
- Real swarm step in the orchestrator (still deferred to whenever LoRA training lands; daily_run.py has a hook for it)
- Failure notifications via separate channel — initial impl logs failures to file; a Phase 7 v1.5 can add a "something broke" notification

## Tasks

### A.1: workers/daily_run.py orchestrator

Discovers clients by scanning `app_data_dir/clients/<id>/credentials_*.json`. For each:
1. Run GSC ingest if `credentials_gsc.json` present
2. Run GA4 ingest if `credentials_ga4.json` present
3. Build dossier
4. (Stub) Run swarm orchestrator dry-run

After all clients: send macOS notification listing how many clients succeeded.

CLI: `python -m ingest.daily_run [--client <id>] [--date YYYY-MM-DD]`. With no args: process all clients for yesterday. Useful for manual triggering.

Important: needs the per-client GSC site URL + GA4 property ID. For Phase 7 thin slice, hardcode those in a `CLIENT_PROPERTIES` dict in `daily_run.py`. Phase 3 v1.5 (per-client config files) will replace it.

### A.2: scripts/install_launchd.sh

Bash script that:
1. Detects the Python venv path (`workers/.venv/bin/python`)
2. Generates the plist with absolute paths
3. Copies to `~/Library/LaunchAgents/`
4. `launchctl load` it

Idempotent: re-running unloads + reloads the plist.

### A.3: scripts/uninstall_launchd.sh

Symmetric: `launchctl unload` + `rm` the plist. Useful if operator wants to pause.

## Verification

- [ ] `python -m ingest.daily_run` runs cleanly, prints per-client status, fires a macOS notification
- [ ] `bash scripts/install_launchd.sh` succeeds; `launchctl list | grep trakauto` shows the agent loaded
- [ ] `~/Library/Logs/trak-automations/daily.log` accumulates lines on each run
- [ ] (Optional manual) `launchctl start com.trakautomations.daily-run` fires the agent immediately, confirms scheduled flow works
